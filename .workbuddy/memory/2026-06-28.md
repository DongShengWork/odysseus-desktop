# 2026年6月28日

## 桌面应用打包功能扩展（参考 OpenCode 项目）

参考 https://github.com/anomalyco/opencode 项目的打包策略，为 Odysseus 桌面项目增加了完整的跨平台打包体系：

### 新增文件
1. **scripts/build-desktop.sh** — 跨平台统一构建入口脚本
   - 自动检测当前操作系统，分发到对应的构建流程（macOS .app/.dmg / Windows .exe / Linux AppImage+.deb+.rpm）
   - 支持 `--clean` `--verbose` `--arch` `--output` `--pkg` 等参数
   - 包含 Linux 图标生成（Pillow/rsvg-convert 自动转换 9 种尺寸）
   - 包含 .desktop 入口、AppRun、DEB 控制文件、RPM SPEC 文件自动生成
   - 构建结束后输出包含产物列表和构建时间的报告

2. **scripts/check-build-health.sh** — 构建健康检查监控脚本
   - 五项独立检查：运行时依赖、构建产物完整性、二进制可执行健康、磁盘空间、项目配置
   - 跨平台兼容：macOS df 命令适配、Linux ELF 二进制检查
   - 状态跟踪：通过 `logs/health/last-status.txt` 持久化检查状态
   - 退出码语义：0=全部通过、1=警告、2=严重失败

3. **scripts/build-orchestrator.sh** — 构建编排器 / 自动化入口
   - 时间感知的循环调度引擎，自主管理 30 分钟检查间隔 + 5分钟重试
   - 状态机：ok → fail → retry → critical（渐进式故障处理）
   - 互斥锁（flock）防并发，最长运行 2 小时后优雅退出
   - 每次自动化触发可执行最多 4 次健康检查

### 定时自动化
- 创建了 HOURLY 定时任务「Odysseus 构建健康监控」
- 编排器在 HOURLY 触发后自主管理 30 分钟间隔检查
- 失败时 5 分钟后自动重试，连续失败则标记 critical 状态

### 跨平台兼容性策略
- macOS：codesign 签名验证、hardened runtime 检测、create-dmg 可选
- Linux：AppImage（linuxdeploy/appimagetool）、DEB（dpkg-deb）、RPM（rpmbuild）
- Windows：原生 PowerShell 路径 + Wine 交叉编译后备方案
- 所有脚本均通过 `bash -n` 语法验证

### 故障修复：macOS venv 权限拒绝（最终根因）

**症状**：双击 Odysseus.app 后提示 "Odysseus failed to start"，日志显示：
```
PermissionError: [Errno 1] Operation not permitted: '.../venv/pyvenv.cfg'
```

**根因分析（三次迭代）**：
1. ❌ `com.apple.provenance` xattr 清理 → 仍报错
2. ✅ **真正根因**: macOS LaunchServices 对**未签名**的 `.app` 应用 App Sandbox 限制。当通过 `open` 命令或 Finder 双击启动时，即使没有 quarantine 属性，LaunchServices 也会限制未签名 app 访问其 bundle 容器之外的文件（包括 venv 目录）。而直接从 Terminal 运行 `bash Odysseus.app/Contents/MacOS/Odysseus` 则完全正常。

**修复**：
```bash
# 无 Apple Developer 账号也可用的 ad-hoc 签名
codesign -s - -f --deep /path/to/Odysseus.app
```

**预防**：`build-macos-app.sh` 已集成 ad-hoc codesign 步骤，每次构建后自动执行。

**鉴别方法**：
- 直接 `bash .../MacOS/Odysseus` 能启动 → 说明 venv 和代码本身没问题
- 通过 `open .../Odysseus.app` 或双击失败 → 说明是 LaunchServices sandbox 问题
- 健康检查脚本已加入签名检测和警告逻辑
